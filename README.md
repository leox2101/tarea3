## Overview

In this task, we developed a piece of code that verifies whether the program is functioning correctly. The verification process focuses on checking the **preconditions** and **postconditions** required for the program to operate as expected.

Additionally, we provide a detailed explanation of the program logic and validate its correctness using formal verification rules.

## Installation

# 1. Prepare opam installation
sudo apt update && sudo apt upgrade
sudo apt install make m4 gcc opam

# 2. opam setup
opam init --disable-sandboxing --shell-setup
eval $(opam env)
opam install -y opam-depext

# 3. Install graphical dependencies
opam depext --install -y lablgtk3 lablgtk3-sourceview3

# 4. Install Frama-C
opam depext --install -y frama-c

# On WSL2, run the GUI as: GDK_BACKEND=x11 frama-c-gui 


## References 


-**Frama-C Project**:
https://frama-c.com
Official site for Frama-C documentation, installation guides, and plugin info.

-**ACSL (ANSI/ISO C Specification Language)**:
https://frama-c.com/html/acsl.html
Language used for annotating C programs with formal contracts.

-**WP Plugin (Weakest Precondition)**:
https://frama-c.com/html/wp.html
Plugin for deductive verification using weakest precondition calculus.

-**Why3 Platform**:
https://why3.lri.fr/
Platform for deductive program verification; Frama-C's WP plugin uses Why3 to interface with automated provers.

-**Alt-Ergo SMT Solver**:
https://alt-ergo.ocamlpro.com/
Automated theorem prover used to discharge goals generated by Why3.
